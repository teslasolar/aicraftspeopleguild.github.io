Design rules

  1. Every expectation has a reason. The reason is the documentation.
  2. Four words only: present, triggered, not triggered, configured.
  3. Three results only: pass, fail, skip.
  4. Tests before code. Write the .test.yml first.
  5. UDT inheritance. Every type traces back to UDT:Expectation.
  6. One test file, one workflow. One test file, one page. The mapping is 1:1.
