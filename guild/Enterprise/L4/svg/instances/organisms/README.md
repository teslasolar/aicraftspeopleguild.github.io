> 🖼 **The organism registry.** Every `.json` file in this directory declares one `SvgOrganism` UDT. They get loaded into `tag.db.udts`, and `svg_build_all.py` iterates the registry on every heartbeat bump, running each organism's `entry` Python script to regenerate its SVG.

## How a new widget gets added

1. Write the generator: `guild/Enterprise/L4/svg/build-<name>.py`. Must expose `main()` that writes one SVG to `guild/Enterprise/L2/hmi/web/assets/svg/<name>.svg`.
2. Drop a JSON instance in this directory: `<name>.json` with `udtType: "SvgOrganism"` and at minimum `parameters.entry`, `parameters.output`, `parameters.size`.
3. Load it into `tag.db.udts` (the tag-DB build step ingests all instance files), or run directly via `python bin/acg build:tag-dbs`.
4. `python bin/acg build:svg-all` — your widget joins the 17 already here and rebuilds on every heartbeat.

No list to maintain. The widget-gallery SVG above is itself generated by iterating this directory.

```json
{
  "udtType": "SvgOrganism",
  "parameters": {
    "name":         "MyWidget",
    "atomicLevel":  "organism",
    "entry":        "guild/Enterprise/L4/svg/build-my-widget.py",
    "output":       "guild/Enterprise/L2/hmi/web/assets/svg/my-widget.svg",
    "sources":      ["url:…", "file:…", "tag:…", "gh:…", "env:…"],
    "trigger_tag":  "demo.heartbeat",
    "composes":     ["WindowChrome", "PulseDot", ...],
    "size":         { "w": 1040, "h": 260 },
    "description":  "…"
  }
}
```
